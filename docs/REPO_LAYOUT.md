# Repository Layout

This document provides a quick overview of where files live and how the repository is structured.

## Top-level

- `README.md`: User-facing install instructions and project overview.
- `docs/`: Documentation for contributors and maintainers.
- `site/`: GitHub Pages content and install scripts.
- `tools/`: Automation scripts used by CI to build and publish packages.

## `site/`

- `index.html.template`: Source template for the landing page.
- `install.sh`: Debian/Ubuntu/Mint installer script (served from Pages).
- `install-rpm.sh`: Fedora/RHEL/CentOS installer script (served from Pages).
- `public/`: Generated by CI to contain repo metadata and published artifacts.

## `tools/`

Scripts are organized by purpose:

- Build + packaging: `build.sh`, `build-prerelease.sh`, `build-and-publish.sh`.
- Repo generation: `generate-apt-repo.sh`, `generate-rpm-repo.sh`.
- Publishing + syncing: `publish-release.sh`, `sync-upstream.sh`, `full-sync-and-build.sh`.
- Diagnostics + validation: `check-upstream.sh`, `validate-repos.sh`, `debug.sh`.
- Setup helpers: `install-apt-repo.sh`, `postinst.sh`, `setup.sh`.

For details on script usage, see `tools/README.md`.
